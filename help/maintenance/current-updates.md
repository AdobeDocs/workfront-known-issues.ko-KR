>[!NOTE]

사용자가 &quot;[!UICONTROL 핀 제거]핀의 &quot; 옵션을 선택하고 제거에 대한 메시지를 받은 다음 &quot;[!UICONTROL 실행 취소]&quot; 메시지에서 핀은 위쪽 탐색에서 바뀌지 않으며 핀의 핀 목록에 추가되지 않습니다 [!UICONTROL 추가 핀] 목록(세 점 메뉴 [!UICONTROL 핀] 영역 참조).

**새로 만든 필터의 이름을 지정하거나 저장할 수 없습니다**

*[!UICONTROL 리소스 플래너]*

**작업 또는 문제의 이름 또는 완료율을 편집할 수 없습니다.**

*작업 및 문제*

사용 중인 사용자 [!UICONTROL Contribute] 작업 또는 문제에 대한 액세스로 헤더에서 작업 또는 문제의 이름을 편집할 수 없습니다. 또한 [!UICONTROL Contribute] 액세스에서는 작업이나 문제의 완료율을 편집할 수 없습니다.

**요청자 및 검토자 수가 조직의 라이선스 수에 포함됨**

*[!DNL Workfront Proof]*

증명에 사용자가 검토자 또는 요청자로 추가되면 “[!UICONTROL 방문자]” 권한 프로필을 받게 되며, 이 경우 [!DNL Workfront Proof] 라이선스를 사용하지 않는 것이 원칙입니다. 그러나 이러한 사용자가 추가되었을 때 사용 [!DNL Workfront Proof] 라이선스의 수가 늘어납니다.

+++**2022년 10월 6일 유지 보수 업데이트**

**새 블루프린트 유형**

*블루프린트*

블루프린트 유형 &quot;대시보드&quot;가 블루프린트 카탈로그에 추가되었습니다. 이전에는 프로젝트 템플릿과 조직 구조 블루프린트만 사용할 수 있었습니다.

**왼쪽 패널에서 겹치는 요소**

*사용자 정의 양식*

사용자가 양식 빌더로 작업할 때 양식의 필드가 100개를 초과하면 필드 제한에 대해 알리는 메시지로 인해 왼쪽 패널의 요소가 겹칩니다.

**날짜 선택기가 입력 포커스에서 더 이상 자동으로 열리지 않거나**

*탐색*

사용자가 키보드를 통해 탐색하면 날짜 선택기가 더 이상 키보드 포커스를 받을 때 자동으로 열리지 않습니다. 대신 키보드 사용자는 날짜 선택기 아이콘으로 탭을 이동하고 Enter 키를 눌러 날짜 선택기를 열어야 합니다. 사용자가 마우스를 탐색할 때 클릭되는 날짜 입력에 따라 날짜 선택기가 더 이상 자동으로 열리지 않습니다. 대신 마우스 사용자는 날짜 선택기 아이콘을 클릭하여 날짜 선택기를 열어야 합니다.

이러한 변경은 표준 날짜 선택기 UX 패턴을 더 잘 준수하고 키보드 및 화면 판독기 사용자가 보다 쉽게 액세스할 수 있는 환경을 만들기 위해 수행되었습니다.

**여러 팀을 할당하면 한 팀만 할당됩니다**

*팀*

>[!NOTE]
>
>이 문제는 미리 보기 환경에만 적용됩니다.

사용자가 여러 팀을 작업이나 문제에 할당하면 하나의 팀만 할당 목록에 나타납니다. 이 문제는 보고에도 영향을 줍니다. 한 팀만 작업이나 문제에 지정된 것으로 나타나므로 팀 할당을 보여 주는 보고서가 부정확합니다.

**&quot;[!UICONTROL 최근 변경 내용이 저장되지 않았습니다]&quot; 작업표에 변경 내용을 자동 저장하는 동안 오류가 발생했습니다.**

*타임시트*

사용자가 자동 저장을 트리거하는 방식으로 타임시트를 편집하려고 하면 변경 내용이 저장되지 않고 다음과 같은 메시지가 표시됨:

&quot;[!UICONTROL 최근 변경 내용이 저장되지 않았습니다. 페이지를 새로 고쳐 봅니다.]&quot;

이 문제는 다음을 편집할 때 보고됨:

* 시간
* 작업

**이메일 알림이 지연됨**

*Workfront 증명*

[!DNL Workfront Proof]에서 이메일 알림을 트리거하는 이벤트가 발생할 때 사용자에게 알림이 즉시 전송되지 않습니다. 알림은 몇 시간 가량 지연될 수 있습니다.

## 2022년 9월 업데이트

+++**2022년 9월 29일 유지 보수 업데이트**

**증명을 닫을 때 사용자가 이전 페이지로 돌아가지 않습니다**

*증명*

내에서 증명을 보고 있는 사용자 [!DNL Workfront] 증명을 닫으면 증명을 열기 전에 있던 페이지가 반환되지 않습니다. 대신, 의 다른 페이지로 리디렉션됩니다 [!DNL Workfront].

**에서 증명을 열 수 없습니다.[!DNL Workfront]**

*증명*

사용자가 [!DNL Workfront] 증명을 열려고 하면 증명이 열리지 않고 사용자가 [!UICONTROL 문서 세부 정보] 페이지.

**사용 시 시간이 저장되지 않습니다 [!UICONTROL 탭] key**

*타임시트*

사용자가 타임시트를 작성하고 [!UICONTROL Tab] 키로 셀 사이를 이동할 때, 시간은 저장되지 않습니다. [!UICONTROL 자동 저장] 알림은 화면 하단에 표시되지 않으며 사용자가 페이지를 새로 고치면 저장되지 않은 시간을 볼 수 있습니다.

**여러 페이지가 있는 증명을 볼 때 빈 페이지**

*[!DNL Workfront Proof]*

사용자가 여러 페이지로 증명을 볼 때 페이지의 축소판을 볼 수 있지만, 기본 뷰어에서는 페이지가 열리지 않습니다.

**문제가 승인 보류 중인 경우 문제 업데이트 스트림이 사라짐**

*업데이트*

사용자가 승인 보류 중인 문제의 업데이트 스트림에서 [!UICONTROL 새 업데이트] 상자를 클릭하면 전체 업데이트 스트림이 사라집니다.

**새 문서 버전을 업로드할 때 오류 발생**

*문서*

사용자가 새 버전의 문서를 업로드하려고 하면 새 버전이 업로드되지 않고 다음 중 한 가지 오류가 표시됩니다.

* [!UICONTROL documentID는 null일 수 없음]
* [!UICONTROL 오류: 잘못된 매개변수: documentID 값이 “정의되지 않음”]

**문서에 대한 공개 링크를 클릭하면 빈 페이지로 연결됨**

*문서*

사용자가 공개 링크를 사용하여 문서를 열려고 하면 빈 페이지로 연결됩니다. 이 문제는 활성 [!DNL Workfront] 세션이 열려 있는 창에서 링크를 열 때 발생합니다.

**대시보드의 보고서에서 목록 제어가 작동하지 않음**

*대시보드*

사용자가 대시보드에서 보고서를 볼 때 보고서의 필터, 그룹화 또는 보기를 변경하려고 하면 필터, 그룹화 또는 보기가 변경되지 않습니다.

**프로젝트를 만들 때 “[!UICONTROL 시스템 관리자여야 합니다.]” 오류 발생**

*프로젝트*

관리자가 아닌 사용자가 프로젝트를 만들고 관리자만 사용 가능한 섹션이 있는 사용자 정의 양식을 첨부할 경우 프로젝트를 만들 수 없고 다음과 같은 오류가 표시됩니다.

“[!UICONTROL 이 사용자 정의 데이터 매개변수 값을 변경하려면 시스템 관리자여야 합니다.]”

**문제를 프로젝트로 전환할 때 사용자 정의 데이터가 유지되지 않음**

*프로젝트*

사용자가 템플릿을 사용하여 문제를 프로젝트로 전환할 때 해당 문제의 사용자 정의 양식 데이터는 프로젝트의 해당 사용자 정의 양식에 전송되지 않습니다. 이 문제는 사용자 정의 필드에 있는 데이터가 다른 사용자 정의 필드의 값에 따라 숨겨질 수 있는 경우에 발생합니다.

**문제를 프로젝트로 전환할 때 오류 발생**

*프로젝트*

사용자가 문제를 프로젝트로 전환하려고 할 때 문제가 전환되지 않고 다음과 같은 오류가 표시됩니다.

“[!UICONTROL 예상치 못한 오류가 발생했습니다.]”

## 2022년 1월 업데이트

+++**2022년 1월 27일 유지 보수 업데이트**

**문제를 프로젝트로 전환할 때 사용자 정의 데이터가 유지되지 않음**

*프로젝트*

사용자가 템플릿을 사용하여 문제를 프로젝트로 전환할 때 해당 문제의 사용자 정의 양식 데이터는 프로젝트의 해당 사용자 정의 양식에 전송되지 않습니다. 이 문제는 사용자 정의 필드에 있는 데이터가 다른 사용자 정의 필드의 값에 따라 숨겨질 수 있는 경우에 발생합니다.

**애자일 보드의 사용자 목록이 알파벳순이 아님**

*애자일*

사용자가 애자일 보드에서 사용자 목록을 볼 때 사용자가 알파벳순으로 표시되지 않습니다. 대신, 할당량이 가장 큰 사용자가 가장 먼저 나열됩니다.

**문제 복사 및 이동 링크 업데이트됨**

*문제*

미리 보기 환경에서 문제 복사 및 이동 링크가 문제 페이지와 문제 목록 모두에서 “[!UICONTROL 복사 위치]” 및 “[!UICONTROL 이동 위치]”로 업데이트되었습니다.

**[!DNL Workfront] 허용 목록에 최대 45개 IP 주소 추가**

*설정*

[!DNL Workfront] 허용 목록에 추가되는 IP 주소 제한을 30개에서 45개로 늘렸습니다.

+++**2022년 1월 20일 유지 보수 업데이트**

**템플릿에서 프로젝트를 만들 때 “[!UICONTROL 잘못된 매개변수]” 오류 발생**

*프로젝트*

사용자가 템플릿에서 프로젝트를 만들고, 프로젝트를 만들 때 템플릿에서 사용자 정의 양식을 제거하려고 하면 프로젝트가 만들어지지 않고 제거된 사용자 정의 양식의 필수 필드를 언급하는 “[!UICONTROL 잘못된 매개변수]” 오류 메시지가 표시됩니다.

**사용자 목록이 [!DNL Safari] 브라우저에 로드되지 않음**

*사용자*

사용자가 [!UICONTROL 사용자] 영역으로 이동하면 헤더가 표시되지만 사용자 목록이 로드되지 않습니다.

**목록에서 작업을 드래그할 때 지연으로 인해 작업이 잘못된 위치로 이동함**

*목록*

사용자가 목록에서 작업을 드래그하여 이동하려고 하면 작업이 이동될 위치를 나타내는 파란색 선이 커서보다 훨씬 느리게 이동합니다. 작업을 드래그하여 놓으면 커서가 목록의 다른 위치를 가리키고 있더라도 작업은 파란색 선 위치로 이동합니다.

+++**[!DNL Workfront Fusion]2022년 1월 14일 유지 보수 업데이트**

**[!UICONTROL 매핑할 새 필드]**&#x200B;를 선택하면 일부 매핑된 필드가 재설정됨

*[!DNL Workfront Fusion]*

[!DNL Workfront] [!UICONTROL 만들기] 또는 [!UICONTROL 업데이트] 모듈의 하나 이상의 필드에서 맵을 활성화하고 매핑할 새 필드를 선택하면 맵이 활성화된 이전에 매핑된 필드의 매핑 값이 손실됩니다.

+++**2022년 1월 13일 유지 보수 업데이트**

**요약 패널에서 하이퍼링크를 댓글에 추가할 수 없음**

*작업*

사용자가 작업 요약 패널에서 댓글을 달고 댓글에 하이퍼링크를 추가하려고 하면 하이퍼링크 창이 열리지만 창을 클릭하면 바로 닫히므로 하이퍼링크를 추가할 수 없습니다. 사용자가 창을 탭한 후 필드에 링크를 입력하거나 붙여넣을 수 있지만 하이퍼링크가 저장되지 않습니다. 두 경우 모두 작업이 선택 해제됩니다.

**팀 편집 페이지가 닫히지 않음**

*팀*

사용자가 팀을 편집할 때 [!DNL Edit team] 페이지가 닫히지 않습니다. 버튼을 클릭하거나, X를 클릭하거나, 페이지에서 벗어나서 페이지를 닫을 수 없습니다.

**이메일 및 인앱 알림이 전송되지 않음**

*알림*

알림을 트리거해야 하는 이벤트가 발생할 때 알림이 전송되지 않습니다. 이 문제는 이메일 또는 인앱 알림에 대해 발생하며, 다른 알림은 전송되더라도 발생할 수 있습니다.

**[!UICONTROL 내 작업] 목록이 비어 있음**

*[!UICONTROL 내 작업]*

사용자가 [!UICONTROL 내 작업] 목록을 볼 때 [!UICONTROL 내 작업] 목록에 대한 레이아웃 템플릿에 [!UICONTROL 완료율]과 같은 숫자 값이 포함되어 있으면 [!UICONTROL 내 작업] 목록이 표시되지 않습니다.

애자일 보드에서 **[!UICONTROL 완료율] 및 [!UICONTROL 완료 시간]을 수정할 수 없음**

*애자일*

사용자가 애자일 보드에서 “[!UICONTROL 작업 항목 더 보기]”를 선택한 다음 새로 로드된 작업 항목 중 하나에서 [!UICONTROL 완료율] 또는 [!UICONTROL 완료 시간]을 변경하려는 경우 완료율 또는 완료 시간을 변경할 수 없습니다. 또한 [!UICONTROL 완료율] 버튼이 회색으로 비활성화됩니다.

+++**2022년 1월 6일 유지 보수 업데이트**

**템플릿 또는 사용자 정의 양식을 프로젝트에 첨부할 때 “[!UICONTROL 잘못된 매개변수]” 오류 발생**

*프로젝트*

사용자가 사용자 정의 양식 또는 템플릿을 기존 프로젝트에 첨부한 다음 사용자 정의 양식 또는 템플릿의 필수 필드를 입력하고 변경 사항을 프로젝트에 저장하려고 하면, 변경 사항이 저장되지 않고 프로젝트 세부 정보 페이지 상단에 “[!UICONTROL 잘못된 매개변수]” 오류가 표시됩니다.

**증명 댓글이 문서 업데이트에 표시되지 않음**

*증명*

사용자가 [!UICONTROL 문서] 영역에서 증명을 볼 때 증명에 대한 댓글이 문서의 [!UICONTROL 업데이트] 영역에 표시되지 않습니다.

**[!UICONTROL 업무 균형자]: “[!UICONTROL ?[오브젝트 오브젝트]?]”가 초과 할당 정보에 표시됨**

*[!UICONTROL 업무 균형자]*

사용자가 사용자의 휴무 일정과 겹치는 작업으로 인해 [!UICONTROL 업무 균형자]에 초과 할당으로 표시되는 경우 다른 사용자가 초과 할당을 조회하면 초과 할당 정보의 “[!UICONTROL 용량]” 영역에 사용자의 실제 용량 대신 “[!UICONTROL ?[오브젝트 오브젝트]?]”가 표시됩니다.
