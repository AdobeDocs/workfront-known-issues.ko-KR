**왼쪽 패널에서 겹치는 요소**

*사용자 정의 양식*

사용자가 양식 빌더에서 작업 중이고 양식에 필드가 100개 이상인 경우, 사용자에게 필드 제한에 대해 알리는 메시지는 왼쪽 패널의 요소가 겹치게 됩니다.

**날짜 선택기가 입력 포커스에서 더 이상 자동으로 열리지 않거나**

*탐색*

사용자가 키보드를 통해 탐색하면 날짜 선택기가 더 이상 키보드 포커스를 받을 때 자동으로 열리지 않습니다. 대신 키보드 사용자는 날짜 선택기 아이콘으로 탭을 이동하고 Enter 키를 눌러 날짜 선택기를 열어야 합니다. 사용자가 마우스를 탐색할 때 클릭되는 날짜 입력에 따라 날짜 선택기가 더 이상 자동으로 열리지 않습니다. 대신 마우스 사용자는 날짜 선택기 아이콘을 클릭하여 날짜 선택기를 열어야 합니다.

이러한 변경은 표준 날짜 선택기 UX 패턴을 더 잘 준수하고 키보드 및 화면 판독기 사용자가 보다 쉽게 액세스할 수 있는 환경을 만들기 위해 수행되었습니다.

**여러 팀을 할당하면 한 팀만 할당됩니다**

*팀*

>[!NOTE]
>
>이 문제는 미리 보기 환경에서만 발생합니다.

사용자가 여러 팀을 작업이나 문제에 할당하면 하나의 팀만 할당 목록에 나타납니다. 이 문제는 보고에도 영향을 줍니다. 한 팀만 작업이나 문제에 지정된 것으로 나타나므로 팀 할당을 보여 주는 보고서가 부정확합니다.

**&quot;[!UICONTROL 최근 변경 내용이 저장되지 않았습니다]&quot; 작업표에 변경 내용을 자동 저장하는 동안 오류가 발생했습니다.**

사용자가 대시보드에서 보고서를 보고 보고서 필터, 그룹화 또는 보기 변경을 시도하면 필터, 그룹화 또는 보기가 변경되지 않습니다.

**&quot;[!UICONTROL 시스템 관리자여야 합니다]&quot; 프로젝트를 만들 때 오류가 발생합니다**

*프로젝트*

관리자가 아닌 사용자가 프로젝트를 만들려고 하고 관리자만 사용할 수 있는 섹션이 있는 사용자 지정 양식을 첨부하면 프로젝트를 만들 수 없으며 다음 오류가 표시됩니다.

&quot;[!UICONTROL 이 사용자 정의 데이터 매개변수 값을 변경하려면 시스템 관리자여야 합니다]&quot;

**문제를 프로젝트로 변환할 때 사용자 지정 데이터가 유지되지 않습니다**

*프로젝트*

사용자가 템플릿을 사용하여 문제를 프로젝트로 변환하면 해당 문제에 대한 사용자 지정 양식의 데이터가 프로젝트의 해당 사용자 지정 양식으로 전송되지 않습니다. 이는 다른 사용자 지정 필드의 값을 기반으로 하여 숨겨질 수 있는 사용자 지정 필드에 있는 데이터에서 발생합니다.

**문제를 프로젝트로 변환하는 동안 오류가 발생했습니다.**

*프로젝트*

사용자가 문제를 프로젝트로 전환하려고 하면 문제가 변환되지 않고 다음 오류가 표시됩니다.

&quot;[!UICONTROL 예기치 않은 오류가 발생했습니다.]&quot;

## 2022년 1월 업데이트

+++**2022년 1월 27일 유지 관리 업데이트**

**문제를 프로젝트로 변환할 때 사용자 지정 데이터가 유지되지 않습니다**

*프로젝트*

사용자가 템플릿을 사용하여 문제를 프로젝트로 변환하면 해당 문제에 대한 사용자 지정 양식의 데이터가 프로젝트의 해당 사용자 지정 양식으로 전송되지 않습니다. 이는 다른 사용자 지정 필드의 값을 기반으로 하여 숨겨질 수 있는 사용자 지정 필드에 있는 데이터에서 발생합니다.

**애자일 보드의 사용자 목록은 알파벳 순서가 아닙니다**

*애자일*

사용자가 애자일 보드에서 사용자 목록을 볼 때 사용자는 알파벳순으로 표시되지 않습니다. 대신 가장 많이 할당된 사용자가 먼저 나열됩니다.

**복사 및 이동 문제에 대한 링크가 업데이트되었습니다**

*문제*

미리 보기 환경에서 복사 및 이동 문제를 위한 링크가 &quot;&quot;로 업데이트되었습니다[!UICONTROL 복사 대상]&quot; 및 &quot;[!UICONTROL 이동 위치]문제 페이지뿐만 아니라 문제 목록에도 있습니다.

**에 최대 45개의 IP 주소를 추가합니다 [!DNL Workfront] 허용 목록에 추가하다**

*설정*

에 추가된 IP 주소의 제한 [!DNL Workfront] 허용 목록에 추가하다가 30개에서 45개로 증가했습니다.

+++

+++**2022년 1월 20일 유지 관리 업데이트**

**&quot;[!UICONTROL 잘못된 매개 변수]&quot; 템플릿에서 프로젝트를 만들 때 오류가 발생했습니다.**

*프로젝트*

사용자가 템플릿에서 프로젝트를 만들려고 하고 프로젝트를 만들 때 템플릿에서 사용자 지정 양식을 제거하면 프로젝트가 생성되지 않고 사용자에게 &quot;&quot;가 표시됩니다[!UICONTROL 잘못된 매개 변수]&quot; 제거된 사용자 지정 양식의 필수 필드를 설명하는 오류 메시지입니다.

**사용자 목록이 로드되지 않음 [!DNL Safari] 브라우저**

*사용자*

사용자가 [!UICONTROL 사용자] 영역이면 머리글이 표시되지만 사용자 목록이 로드되지 않습니다.

**목록에서 작업을 드래그할 때 지연이 발생하면 작업이 잘못된 위치로 이동합니다**

*목록*

사용자가 목록에서 작업을 끌어 이동하려고 하면 작업이 이동할 위치를 나타내는 파란색 선이 커서보다 훨씬 느리게 이동합니다. 사용자가 작업을 해제하면 커서가 목록의 다른 위치를 가리키더라도 파란색 선이 있는 위치로 이동합니다.

+++

+++**[!DNL Workfront Fusion]2022년 1월 14일 유지 관리 업데이트**

**선택할 때 일부 매핑된 필드가 재설정됩니다. [!UICONTROL 매핑할 새 필드]**

*[!DNL Workfront Fusion]*

에 하나 이상의 필드가 있는 경우 [!DNL Workfront] [!UICONTROL 만들기] 또는 [!UICONTROL 업데이트] 모듈에는 맵이 활성화되어 있고 사용자가 매핑할 새 필드를 선택하면 맵이 활성화된 이전에 매핑된 필드의 매핑 값이 손실됩니다.

+++

+++**2022년 1월 13일 유지 관리 업데이트**

**요약 패널에서 주석에 하이퍼링크를 추가할 수 없습니다**

*작업*

사용자가 작업의 요약 패널에서 주석을 작성하고 주석에 하이퍼링크를 추가하려고 하면 하이퍼링크 창이 열리지만 사용자가 창으로 클릭하는 즉시 닫히고 하이퍼링크를 추가할 수 없습니다. 사용자가 창을 탭하면 필드에 링크를 입력하거나 붙여넣을 수 있지만 하이퍼링크는 저장되지 않습니다. 두 경우 모두 작업이 선택 취소됩니다.

**팀 편집 페이지가 닫히지 않음**

*팀*

사용자가 팀을 편집하려고 하면 [!DNL Edit team] 페이지가 닫히지 않습니다. 사용자는 버튼을 클릭하거나 X를 클릭하거나 페이지에서 멀리 탐색하여 페이지를 닫을 수 없습니다.

**이메일 및 인앱 알림이 전송되지 않습니다**

*알림*

알림을 트리거해야 하는 이벤트가 발생하면 알림이 전송되지 않습니다. 이는 이메일 또는 인앱 알림에 대해 발생할 수 있으며 다른 알림이 전송되고 있더라도 발생할 수 있습니다.

**[!UICONTROL 내 작업] 목록이 비어 있음**

*[!UICONTROL 내 작업]*

사용자가 자신의 [!UICONTROL 내 작업] 목록 및 레이아웃 템플릿의 [!UICONTROL 내 작업] 목록에는 다음과 같은 숫자 값이 포함되어 있습니다. [!UICONTROL 완료율], [!UICONTROL 내 작업] 목록이 표시되지 않습니다.

**[!UICONTROL 완료율] 및 [!UICONTROL 완료 시간] 애자일 보드에서 수정할 수 없음**

*애자일*

사용자가 &quot;[!UICONTROL 작업 항목 더 표시]&quot;(Agile Board)에서 [!UICONTROL 완료율] 또는 [!UICONTROL 완료 시간] 새로 로드된 작업 항목 중 하나에서 완료 퍼센트 또는 완료 시간을 변경할 수 없습니다. 다음 [!UICONTROL 완료율] 단추도 회색으로 표시되어 비활성 상태임을 나타냅니다.

+++**2022년 1월 6일 유지 관리 업데이트**

**&quot;[!UICONTROL 잘못된 매개 변수]&quot; 템플릿 또는 사용자 지정 양식을 프로젝트에 첨부할 때 오류가 발생했습니다.**

*프로젝트*

사용자가 사용자 지정 양식이나 템플릿을 기존 프로젝트에 첨부하려고 하면 사용자 지정 양식이나 템플릿의 필수 필드를 채워 프로젝트에 변경 사항을 저장하면 변경 사항이 저장되지 않고 사용자에게 &quot;[!UICONTROL 잘못된 매개 변수]&quot;라는 오류가 프로젝트 세부 사항 페이지의 맨 위에 있습니다.

**문서 업데이트에 증명 주석이 표시되지 않습니다**

*증명*

사용자가 [!UICONTROL 문서] 영역, 증명 자체에 대해 수행된 모든 주석이 [!UICONTROL 업데이트] 문서의 영역.

**[!UICONTROL 업무 균형자]: &quot;[!UICONTROL ?[개체]?]&quot;은 초과 할당 정보를 표시합니다.**

*[!UICONTROL 업무 균형자]*

사용자가 [!UICONTROL 작업 로드 밸런서] 사용자의 시간이 겹치는 작업과 다른 사용자가 초과 할당을 보고 있으므로 &quot;[!UICONTROL 용량]&quot; 초과 할당 정보 영역에 &quot;[!UICONTROL ?[개체]?]&quot; 실제 용량을 할당하는 것이 좋습니다.
