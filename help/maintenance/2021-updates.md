# 2021년 [!DNL Workfront] 유지 관리 업데이트

2021년에 다음과 같은 유지 관리 업데이트가 수행되었습니다.

## 2021년 12월 업데이트

+++**2021년 12월 23일 유지 관리 업데이트**

**새 작업은 기본적으로 잘못된 작업 제한으로 설정됩니다.**

_Tasks_

사용자가 &quot;[!UICONTROL 새 작업]&quot; 단추와 [!UICONTROL 새 작업 기본 시작] 날짜 옵션이 &quot;[!UICONTROL 오늘],&quot; 생성된 작업의 작업 제한이 &quot;[!UICONTROL 가능한 한 빨리]&quot;[!UICONTROL 보다 빨리 시작].&quot; 프로젝트 템플릿을 사용할 때도 이 문제가 발생할 수 있습니다.

**예약 시작 위치 [!UICONTROL 그룹] 영역으로 인해 빈 페이지가 표시됩니다**

_설정_

사용자가 [!UICONTROL 설정] 영역 및 예약 열기, 편집 또는 복사를 시도하면 일정이 열리지 않고 사용자에게 빈 페이지가 표시됩니다.

**왼쪽 탐색 순서를 변경할 때 변경 사항이 표시되지 않습니다**

_탐색_

사용자가 항목을 드래그하여 왼쪽 탐색 패널의 순서를 변경하려고 하면 항목이 사용자가 해당 항목을 삭제하면 이전 위치로 다시 나타납니다. 사용자가 페이지를 새로 고치면 왼쪽 패널에 새 순서가 표시됩니다.

**링크가 요청된 문서를 제출하면 빈 페이지가 표시됩니다.**

_문서_

사용자가 문서 제출 요청을 받고 문서가 요청된 객체에 대한 링크를 클릭하면 링크가 빈 페이지로 이동합니다. 이 문제는 이메일이나 인앱 알림에서 링크를 클릭할 때 발생할 수 있습니다.

**[!UICONTROL 작업 로드 밸런서] 할당된 0시간 표시**

_[!UICONTROL 업무 균형자]_

사용자가 [!UICONTROL 작업 로드 밸런서] 및 에는 &quot;[!UICONTROL 예상 날짜 표시]&quot; 설정이 활성화되면 이후의 모든 날짜에 할당된 0시간이 표시됩니다.

**증명이 폴더에서 간헐적으로 사라집니다**

_[!DNL Workfront Proof]_

사용자가 로그인한 경우 [!DNL Workfront Proof] 폴더를 보면 빈 폴더가 표시됩니다. 사용자가 나중에 다시 확인하면 증명이 표시됩니다.

+++**2021년 12월 16일 유지 관리 업데이트**

**알림 목록에서 알림을 클릭하면 빈 페이지가 표시됩니다**

_알림_

사용자가 다음에서 알림 목록을 열면 [!UICONTROL 알림 을 참조하십시오] 아이콘 을 클릭한 다음 공지 사항을 클릭하면 빈 페이지로 이동하며 공지 사항이 표시되지 않습니다.

**요약 패널에 &quot;[!UICONTROL 선택 안 함]&quot; 작업을 선택하면**

_작업_

사용자가 [!UICONTROL 문서] 프로젝트의 영역으로 이동한 다음 작업 목록으로 이동하여 작업을 선택하고 작업 요약을 열려고 하면 작업 요약이 표시되지 않고 사용자에게 다음 메시지가 표시됩니다.

[!UICONTROL 선택 사항 없음. 목록에서 문서를 선택하여 세부 정보를 확인합니다.]

사용자가 작업 목록에 있어도 메시지에 문서에 대한 설명이 표시됩니다.

**[!UICONTROL 할당되지 않은 작업] 로드되지 않음**

_[!UICONTROL 업무 균형자]_

사용자가 [!UICONTROL 작업 로드 밸런서] 를 사용하여 필터 만들기 [!UICONTROL 배정:역할 ID] 필드, [!UICONTROL 할당되지 않은 작업] 영역이 로드되지 않습니다.

**&quot;을(를) 사용하여 템플릿 첨부[!UICONTROL 사용자 지정 및 첨부]&quot; 옵션은 사용자 지정 필드 값을 지웁니다.**

_프로젝트_

사용자가 &quot;[!UICONTROL 사용자 지정 및 첨부]&quot; 옵션을 사용할 수 있으며 프로젝트에 이미 사용자 지정 양식이 첨부되어 있습니다. 사용자 지정 필드 값은 전달되지 않으며 수동으로 다시 입력해야 합니다. 이 문제는 템플릿에 동일한 사용자 지정 양식이 포함되어 있는 경우에도 발생합니다.

_승인_

_보고서_

사용자가 보고서의 할당을 인라인 편집하려고 하면 필드 값이 변경되지 않고 편집은 저장되지 않습니다.

_탐색_

**할당되지 않은 작업이 로드되지 않음**

## 2021년 11월 업데이트

+++**2021년 11월 18일 유지 관리 업데이트**

**[!DNL Workfront]대상 [!DNL Jira] &quot;[!UICONTROL clientID 또는 clientSecret이 잘못되었습니다.]로그인 시 &quot; 오류**

_Workfront 통합_

사용자가 [!DNL Jira] Workfront 통합을 위해 사용할 수 있습니다. 사용자가 [!DNL Workfront for Jira] 통합하면 로그인할 수 없으며 다음 오류가 표시됩니다.

&quot;[!UICONTROL clientID 또는 clientSecret이 잘못되었습니다.]&quot;

**새 큐 항목을 선택하면 요청에 첨부된 사용자 지정 양식이 업데이트되지 않습니다**

_요청_

사용자가 요청을 만들고 요청에 사용자 지정 양식을 자동으로 첨부하는 큐 항목을 선택한 다음 다른 큐 항목을 선택하면 첫 번째 큐 항목의 사용자 지정 양식이 요청에 첨부되어 유지됩니다.

**아이콘이 잘못 표시됩니다.**

_[!DNL Workfront]_

아이콘 이미지가 잘못 표시됩니다. 이 기능은 여러 지역에서 보고되었습니다 [!UICONTROL Workfront].

**&quot;기타 크기&quot; 옵션이 선택된 경우 작업이 PDF으로 내보내지지 않습니다.**

_작업_

사용자가 작업 목록을 PDF으로 내보내려고 하고 &quot;[!UICONTROL 기타 크기]&quot; 옵션을 선택하면 크기를 선택하고 [!UICONTROL 내보내기]로 설정되지는 않지만, 목록을 내보내지 않습니다. 오류 메시지가 없으며 내보내기가 실패했음을 나타내는 다른 표시도 없습니다.

**이미지 표시기가 전자 메일 알림에 표시되지 않습니다**

_알림_

사용자가 업데이트를 통해 이미지를 추가하고 업데이트를 받는 사람에게 이메일 알림이 전송되면 업데이트에는 업데이트에 이미지가 있다는 표시기가 전자 메일에 포함되지 않습니다.

**특정 브라우저에서 페이지가 로드되지 않음**

_[!DNL Workfront]_

사용자가 작업 중인 경우 [!DNL Workfront], 페이지가 로드되지 않고 사용자에게 다음 오류 메시지가 표시됩니다.

&quot;[!UICONTROL 오류가 발생했으며 문제를 해결하기 위해 노력 중입니다. 작업을 계속하려면 이 브라우저 페이지를 새로 고침해 보십시오.]&quot;

* [!DNL Firefox]
* [!DNL Microsoft Edge]

+++**2021년 11월 11일 유지 관리 업데이트**

**문서 통합 문제, 문서 업로드 팝업에서 빈 페이지[!DNL Google Drive*]*

_문서_

사용자가 새 문서를 [!DNL Workfront] 변환 전: [!DNL Google Drive] 사용 [!UICONTROL 새로 추가] >[!UICONTROL From [!DNL Google Drive]]로 설정되면 업로드 팝업 화면이 공백으로 유지됩니다.

**작업 로드 밸런서에서 필터를 두 개 이상 사용할 수 없습니다.**

_[!UICONTROL 업무 균형자]_

사용자가 [!UICONTROL 작업 로드 밸런서]에는 다음 문제가 표시됩니다.

* 사용자가 두 개의 필터를 선택하면 아래쪽 필터만 적용됩니다.
* 사용자가 두 개 이상의 필터를 선택하면 결과가 표시되지 않습니다.

**&quot;[!UICONTROL 프로젝트 폴더]프로젝트 문서 영역에 문서 헤더가 없습니다.**

_프로젝트_

사용자가 프로젝트에 있고 프로젝트의 문서를 보면 &quot; 제목이 표시됩니다.[!UICONTROL 프로젝트 폴더]&quot;&quot;이(가) 왼쪽 탐색에서 누락되었습니다. 드롭다운 화살표가 여전히 있으며 사용자가 폴더를 선택할 수 있습니다.

**간판 보드의 열이 너무 커서 조정할 수 없습니다**

_애자일_

사용자가 몇 개의 열이 있는 간판 보드를 볼 때 열이 너무 넓고, 사용자는 스크롤하여 카드를 보거나 가장 오른쪽 열로 이동해야 합니다. 열 너비는 조정할 수 없으므로 사용자는 열을 작게 만들 수 없으므로 동시에 화면에서 모두 볼 수 있습니다.

**새 팀을 만들기 위한 인터페이스가 개선되었습니다.**

_팀_

이제 새로운 시각적 큐를 통해 팀을 만드는 것이 보다 직관적입니다. 을(를) 선택하는 경우 [!UICONTROL 팀 전환] 아이콘 사용 [!UICONTROL 새 팀 만들기] 링크에는 &quot;[!UICONTROL 새],&quot;로 설정되고 링크와 다른 이름이 구분되므로 팀 이름이 아닌 것처럼 보입니다. 이 인터페이스는 민첩하고 민첩하지 않은 팀에 대해 동일합니다.

+++**2021년 11월 4일 유지 관리 업데이트**

**새 작업은 기본적으로 잘못된 작업 제한으로 설정됩니다.**

_작업_

사용자가 &quot;[!UICONTROL 새 작업]&quot; 버튼을 누르고 새 작업 기본 시작 날짜 옵션이 &quot;[!UICONTROL 오늘],&quot; 생성된 작업의 작업 제한이 &quot;[!UICONTROL 가능한 한 빨리]&quot;[!UICONTROL 보다 빨리 시작].&quot;

**필드가 Agile 스토리 카드에 표시되지 않습니다**

_애자일_

사용자가 Agile 스토리보드를 볼 때 카드에는 [!UICONTROL 설명] 및 [!UICONTROL 상태] 필드. 사용자 지정 필드를 포함한 다른 모든 필드는 표시되지 않습니다.

**새 열로 이동하기 전에 카드가 원래 열로 돌아갑니다**

_애자일_

사용자가 카드를 스토리보드의 새 열로 드래그하면 카드를 드래그하는 것을 볼 수 있습니다. 그러나 사용자가 카드를 새 열에 놓으면 카드가 새 열에 표시되기 전에 원래 열에 간단히 나타납니다.

**필터의 사용자 지정 필드에 값을 사용할 수 없습니다.**

**전자 메일 알림 설정을 업데이트할 수 없습니다.**

_설정_

사용자가 이메일 알림에 대한 설정을 업데이트하려고 하면 [!UICONTROL 이메일 알림] 영역 및 &quot; 오류 메시지를 참조하십시오.[!UICONTROL 다시 한번 해봅시다 이런! 문제가 발생했습니다. 연락처 [!DNL Workfront] 그래서 우리는 무엇이 잘못되었는지 알아내고 그것을 고칠 수 있습니다.]&quot;

**[!UICONTROL 간트 차트] 일부 필드가 잘리게 합니다.**

_목록_

사용자가 [!UICONTROL 간트 차트] 일부 목록 영역에서 [!UICONTROL 설명]- 텍스트를 자릅니다. 전체 텍스트를 보려면 필드를 두 번 클릭해야 합니다.

**에서 파일을 보낼 수 없습니다. [!UICONTROL 문서 세부 정보]**

_문서_

사용자가 새 [!DNL Workfront] experience manager는 [!UICONTROL 문서 세부 정보] 페이지에 오류 메시지가 표시됩니다.[!UICONTROL 오류가 발생하여 문제를 해결하기 위해 노력하고 있습니다. 작업을 계속하려면 이 브라우저 페이지를 새로 고쳐 보십시오.]&quot;

+++**2021년 1월 7일 유지 관리 업데이트**

**승인 위임 대화 상자가 닫힙니다**

_홈_

사용자가 승인을 위임하려고 할 때 [!UICONTROL 홈] 아무 날짜나 클릭하면 날짜를 선택하거나 사용자가 사용자 위임을 완료할 수 없도록 대화 상자가 닫힙니다.

**문서를 작업으로 이동하는 문제**

_문서_

사용자가 새 문서에서 문서나 증명을 이동하려고 할 때 [!DNL Workfront] 경험, 프로젝트 외부의 일부 작업은 상위 프로젝트를 예상대로 나열하지 않습니다.

**다운로드한 PDF의 이름이 잘못되었습니다**

_[!DNL Workfront Proof]_

사용자가 이메일([!UICONTROL 증명] > [!UICONTROL 주석 인쇄] > [!UICONTROL PDF])를 사용하고 파일을 내보내면 다운로드한 파일의 제목은 증명 ID 대신 난수로 지정됩니다.
